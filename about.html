About This Portfolio Site

This page documents how my website was built. The stack, deployment, interactive features, and the journey from a simple HTML page to a full personal portfolio.

01 What This Site Is

This is my personal website hosted on GitHub Pages - a static site that showcases my education, experience, projects, hobbies, and ways to connect with me. It doubles as a living project demonstrating web development, UI design, and integration with a real database for visitor interaction.

Unlike a template-based site builder, this was hand-coded so every section, animation, and feature could be customized to reflect my work and personality.

03 How It Works

GitHub Pages serves plain HTML, CSS, and JavaScript files directly from the repository. There is no server-side code running on GitHub's end. This means traditional databases can't connect directly. That's where Supabase comes in.

Static Pages

Each page (Home, Projects, UniSwap, etc.) is a standalone HTML file styled with css/main.css.

Shared Components via JS

js/site.js injects the navigation bar, footer, and animated background on every page.

Theme System

js/theme.js toggles dark/light mode using CSS variables and saves your preference in localStorage.

Animations

CSS keyframes power the breathing background orbs. js/animations.js handles scroll-reveal effects.

Click Effects

js/interactions.js adds ripple effects when you click links and buttons.

Feedback & Q&A

js/feedback.js talks to Supabase to store ratings, written reviews, and visitor questions.

04 Interactive Features

Reviews & Ratings

The Reviews page lets visitors rate the portfolio and leave written feedback. Each major project (UniSwap, Weather App, Currency Calculator) also has its own rating widget with a comment box. Data is stored in a Supabase feedback table with star ratings and optional messages.

Q&A Section

Visitors can submit questions on the Reviews page. I answer them in the Supabase dashboard, mark them as published, and they appear on the site. This creates a lightweight FAQ without needing a custom admin panel.

Dark / Light Mode

The toggle in the navigation bar switches between a dark tech theme (default) and a light mode. All colors are defined as CSS custom properties in css/main.css, so the entire site transitions smoothly.

06 Deployment

The site deploys automatically through GitHub Pages. When I push changes to the main branch, GitHub rebuilds and publishes the site to josephn04.github.io. No server configuration, Docker, or CI pipeline is required for the static files.

Supabase runs separately as a cloud service. The frontend connects to it over HTTPS using the public anon key, with Row Level Security policies controlling what visitors can read and write.

07 The Journey

This site all started as a basic HTML/CSS course assignment and I decided to evolve it through several redesigns:

  • Version 2 - Modern card layout, improved typography, and mobile responsiveness.
  • Version 3 (current) - Dark tech theme, animated backgrounds, Supabase-powered feedback, per-project ratings, Q&A, dedicated UniSwap capstone page, resume download, and light/dark mode.
